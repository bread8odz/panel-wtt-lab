# panel-wtt-lab

## 项目简介

本仓库主要用于归档和发布多个独立的 HTML 页面。这些页面通常用于测试、演示、原型展示或作为特定场景下的静态界面使用。

仓库本身不针对任何特定域名或网站，所有页面均为独立模块，可单独访问或引用。

## 目录结构

```
├── pages/            # 存放所有 HTML 页面文件
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/           # 页面可能用到的静态资源（CSS、JS、图片等）
├── README.md         # 本文件
└── LICENSE           # 仓库许可证
```

- `pages/`：所有独立 HTML 页面存放于此，文件名通常体现页面用途或功能。
- `assets/`：存放页面共用的样式、脚本或图片资源，便于集中管理。

## 页面归档说明

每个 HTML 页面均为独立完整文件，可直接在浏览器中打开运行。页面之间无强依赖关系，但部分页面可能引用 `assets/` 下的公共资源。

页面归档遵循以下原则：

- 每个页面文件保持自包含性，减少外部依赖。
- 页面命名清晰，易于识别其用途。
- 如需更新或替换某个页面，直接覆盖对应文件即可。

## 维护说明

- 本仓库由维护者不定期更新，用于新增或归档新的 HTML 页面。
- 若页面涉及外部资源（如 CDN 库、字体等），请确保引用地址稳定可靠。
- 提交新页面时，建议在 `pages/` 目录下添加对应文件，并在必要时更新本 README 的描述。
- 如有页面废弃或不再维护，可将其移入 `archive/` 目录（若存在），或直接删除并记录在提交信息中。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 使用浏览器直接打开 `pages/` 目录下的任一 HTML 文件。
3. 也可将页面部署到任意静态服务器上，通过 URL 直接访问。
